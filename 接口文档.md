# Ragent 项目接口文档

## 1. 认证模块

### 1.1 登录接口
- **路径**: `/api/ragent/auth/login`
- **方法**: `POST`
- **功能**: 用户登录
- **请求体**:
  ```json
  {
    "username": "admin",
    "password": "admin"
  }
  ```
- **响应**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "操作成功",
    "data": {
      "token": "xxxxx",
      "user": {
        "id": "2001523723396308993",
        "username": "admin",
        "role": "admin",
        "avatar": "https://static.deepseek.com/user-avatar/G_6cuD8GbD53VwGRwisvCsZ6"
      }
    }
  }
  ```

### 1.2 登出接口
- **路径**: `/api/ragent/auth/logout`
- **方法**: `POST`
- **功能**: 用户登出
- **响应**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "操作成功"
  }
  ```

## 2. RAG 核心模块

### 2.1 流式对话接口
- **路径**: `/api/ragent/rag/v3/chat`
- **方法**: `GET`
- **功能**: 发起 SSE 流式对话
- **参数**:
  - `question`: 问题内容
  - `conversationId` (可选): 会话 ID
  - `deepThinking` (可选): 是否启用深度思考模式，默认 false
- **响应**: SSE 流式输出

### 2.2 停止任务接口
- **路径**: `/api/ragent/rag/v3/stop`
- **方法**: `POST`
- **功能**: 停止指定任务
- **参数**:
  - `taskId`: 任务 ID
- **响应**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "操作成功"
  }
  ```

## 5. 文档管理模块

### 5.1 上传文档
- **路径**: `/api/ragent/knowledge-base/{kb-id}/docs/upload`
- **方法**: `POST`
- **功能**: 上传文档到知识库
- **参数**:
  - `file`: 文件（multipart/form-data）
  - `docName`: 文档名称
  - `processMode`: 处理模式（chunk 或 pipeline）
  - `chunkStrategy`: 分块策略
- **响应**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "操作成功",
    "data": {
      "id": "2019941957409554433",
      "docName": "RAG 架构设计",
      "fileUrl": "http://localhost:9000/docs/rag-arch.pdf",

    }
  }
  ```

## 9. 响应格式说明

所有接口的响应格式统一为：

```json
{
  "success": true,          // 是否成功
  "code": 200,              // 响应码
  "message": "操作成功",     // 响应消息
  "data": {}               // 响应数据（可选）
}
```

### 响应码说明
- `200`: 操作成功
- `400`: 请求参数错误
- `401`: 未授权
- `403`: 权限不足
- `404`: 资源不存在
- `500`: 服务器内部错误

## 10. 认证说明

- 所有需要认证的接口都需要在请求头中携带 `Authorization` 字段
- 登录成功后，服务器会返回 `token`，将其放入 `Authorization` 头中
- 格式：`Authorization: {token}`

## 11. 接口前缀

所有接口的基础路径为：`/api/ragent`

例如：`/auth/login` 完整路径为 `/api/ragent/auth/login`

## 12. 注意事项

1. **文件上传**：文件上传接口使用 `multipart/form-data` 格式
2. **流式接口**：`/rag/v3/chat` 接口使用 SSE（Server-Sent Events）格式
3. **幂等性**：部分接口使用了 `@IdempotentSubmit` 注解，确保操作幂等
4. **限流**：系统对并发请求有一定限制，超过限制会排队处理
5. **错误处理**：所有错误都会返回统一的错误格式，包含错误码和错误消息
