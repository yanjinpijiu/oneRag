# 知识库重构需求分析与接口文档

## 1. 需求分析

### 1.1 目标
- 文件上传能力迁移到“知识库”模块。
- 上传与处理解耦：先上传到 S3，后手动触发分片/向量化/入 Milvus。
- 支持文件管理：新增、删除、分页查询、下载。
- 支持状态管理：`UNPROCESSED`、`RUNNING`、`COMPLETED`、`FAILED`。

### 1.2 权限
- 登录用户：可查看列表、查看详情、获取下载链接。
- 管理员：额外可上传、触发处理、删除。

### 1.3 一致性策略
- 上传成功后，立即落库 `UNPROCESSED`。
- 处理失败时，不删除 S3 原文件，仅回滚 Milvus 分片与向量数据，状态更新为 `FAILED` 并记录错误，可重试。
- 删除文档时同时删除 S3 对象与 Milvus 向量，再逻辑删除数据库记录。

## 2. 数据模型

表：`kb_document`
- `document_id`：业务文档 ID（UUID）
- `file_name`、`file_ext`、`mime_type`、`file_size`
- `s3_bucket`、`s3_object_key`、`file_url`
- `status`、`chunk_count`、`vector_count`、`error_message`
- `created_by`、`updated_by`、`create_time`、`update_time`、`deleted`

建表脚本：`src/main/resources/sql/create_kb_document_table.sql`

## 3. 接口文档（V1）

### 3.1 分页查询
- `GET /api/kb/documents?pageNo=1&pageSize=10&status=UNPROCESSED&keyword=xxx`
- 鉴权：登录
- 返回：分页列表

### 3.2 文档详情
- `GET /api/kb/documents/{documentId}`
- 鉴权：登录
- 返回：文档元数据与状态

### 3.3 上传文件（仅上传）
- `POST /api/kb/documents/upload`
- Content-Type: `multipart/form-data`
- 参数：`file`
- 鉴权：管理员
- 行为：上传 S3，记录 `UNPROCESSED`

### 3.4 手动触发处理
- `POST /api/kb/documents/{documentId}/process`
- 鉴权：管理员
- 请求体（可选）：
```json
{
  "strategy": "fixed-size",
  "chunkSize": 500,
  "overlapSize": 50
}
```
- 行为：解析 -> 分片 -> 向量化 -> 写 Milvus -> 更新状态

### 3.5 删除文档
- `DELETE /api/kb/documents/{documentId}`
- 鉴权：管理员
- 行为：删除 S3 + 删除 Milvus + 逻辑删 DB

### 3.6 下载链接
- `GET /api/kb/documents/{documentId}/download-url`
- 鉴权：登录
- 返回：预签名 URL + 过期时间（秒）

## 4. 方案改进建议
- Milvus 字段命名建议后续统一（当前使用 `metadata.doc_id` 做文档级删除）。
- 删除操作建议后续补充异步补偿任务，增强跨系统一致性。
- 处理流程可后续升级为异步任务队列，提升大文件处理稳定性。
