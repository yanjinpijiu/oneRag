# Onerag 前后端联调说明

## 1. 启动顺序

### 1.1 初始化数据库
1. 执行会话表脚本：`src/main/resources/sql/create_conversation_tables.sql`
2. 执行用户表脚本：`src/main/resources/sql/create_user_table.sql`
3. 按脚本注释插入管理员账号（替换为真实密码哈希）

### 1.2 启动后端
在项目根目录执行：

```bash
mvn spring-boot:run
```

后端默认地址：`http://localhost:8080`

### 1.3 启动前端
在 `frontend` 目录执行：

```bash
npm install
npm run dev
```

前端默认地址：`http://localhost:5173`

> 已配置 Vite 代理 `/api -> http://localhost:8080`。

## 2. 页面与路由

- `/login`：登录/注册
- `/chat`：聊天页面（SSE 流式）
- `/knowledge`：知识库
- `/users`：用户管理（仅 admin 显示与可访问）
- `/profile`：个人中心

## 3. 鉴权约定

- 登录成功后，前端把 token 保存在 `localStorage.token`
- 所有接口自动带 `Authorization` 请求头
- 后端使用 Sa-Token，管理员角色通过 `role=admin` 判定
- `/api/chat/**` 与 `/api/document/**` 已接入登录校验，未登录会返回鉴权错误

## 4. 当前实现说明

- 聊天页已对接会话管理与流式接口：
  - `GET /api/chat/conversations`：按当前登录用户查询历史会话列表
  - `GET /api/chat/conversations/{conversationId}/messages`：查询指定会话消息历史
  - `DELETE /api/chat/conversations/{conversationId}`：删除当前用户自己的会话
  - `POST /api/chat/stream`：发送消息并流式接收回复
- 聊天页支持：
  - 左侧历史会话列表、会话切换、新建会话、删除会话
  - `meta/queryRewrite/intent/reasoning/content/references/done/error` 事件解析
  - 对话ID记忆（`sessionStorage`），切页后返回可恢复历史
  - 深度思考开关
- 会话鉴权规则：
  - 会话列表仅返回当前登录用户数据
  - 查询/删除会话会校验归属，不能访问他人会话
- 知识库页已支持：
  - 文件分页查询（登录可读）
  - 管理员上传文件（先入 S3，状态为 `UNPROCESSED`）
  - 管理员手动触发处理（分片、向量化、入 Milvus）
  - 管理员删除文件（联动删除 S3 + Milvus + DB）
  - 登录用户下载文件（预签名 URL）
- 用户管理页支持分页查询、新增、编辑、删除
- 个人中心支持资料修改与密码修改
- 聊天接口与上传接口均按登录用户写入/校验（不再固定 `default` 用户）

## 5. 常见问题

- 401/未登录：
  - 检查是否先登录
  - 检查请求头 `Authorization` 是否携带 token
- 无权限访问用户管理：
  - 当前账号必须为 `admin`
- 上传失败：
  - 检查后端 `spring.servlet.multipart` 配置是否生效
  - 检查代理和后端端口
