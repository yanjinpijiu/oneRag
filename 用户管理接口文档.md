# Onerag 用户管理模块接口文档

## 1. 需求分析

### 1.1 业务目标
- 在现有问答与文档上传系统基础上，补齐用户体系，支持账号登录后访问系统能力。
- 增加后台用户管理能力（新增、编辑、删除、查询）。
- 提供个人中心能力（查看/更新资料、修改密码）。
- 前端重建为 Vue3 工程，覆盖登录、聊天、文件上传、用户管理、个人中心。

### 1.2 功能范围
- 认证：注册、登录、登出、当前用户信息。
- 用户管理：用户分页查询、创建、更新、删除、详情。
- 个人中心：资料更新、密码修改。
- 鉴权：未登录不可访问受保护接口；仅管理员可访问用户管理接口。

### 1.3 技术约束
- 后端：Spring Boot + MyBatis-Plus + Sa-Token。
- 前端：Vue3 + Vite + Vue Router + Pinia + Element Plus。
- 数据库：MySQL，沿用逻辑删除与审计字段风格。

## 2. 数据库设计与 SQL

> 你当前已有 `t_conversation`、`t_conversation_message`。新增用户表如下。

```sql
create table t_user
(
    id              bigint                                  not null comment '主键ID'
        primary key,
    user_id         varchar(36)                             not null comment '用户ID(UUID)',
    username        varchar(64)                             not null comment '登录账号',
    password_hash   varchar(255)                            not null comment '密码哈希',
    nickname        varchar(64)                             not null comment '昵称',
    email           varchar(128)                            null comment '邮箱',
    phone           varchar(32)                             null comment '手机号',
    avatar          varchar(512)                            null comment '头像URL',
    status          tinyint       default 1                 not null comment '状态:1启用,0禁用',
    role            varchar(32)   default 'user'            not null comment '角色:user/admin',
    last_login_time datetime                                null comment '最后登录时间',
    create_time     datetime                                not null comment '创建时间',
    update_time     datetime                                not null comment '更新时间',
    deleted         int           default 0                 null comment '删除标识',
    constraint uk_user_user_id
        unique (user_id),
    constraint uk_user_username
        unique (username),
    constraint uk_user_email
        unique (email),
    constraint uk_user_phone
        unique (phone)
)
    comment '用户表' charset = utf8mb4;

create index idx_user_status
    on t_user (status);

create index idx_user_role
    on t_user (role);

create index idx_user_create_time
    on t_user (create_time);
```

管理员初始化（示例）：


## 3. 统一响应约定

- code: `"0"` 表示成功，其它为失败。
- message: 人类可读信息。
- data: 业务数据。
- requestId: 可选，链路追踪用。

## 4. 鉴权说明

- 采用 Sa-Token。
- 登录成功后返回 `token`，前端后续请求放在请求头 `Authorization`。
- 管理员接口需 `role=admin`。

## 5. 认证接口

### 5.1 注册
- 方法：`POST`
- 路径：`/api/auth/register`
- 请求体：

```json
{
  "username": "alice",
  "password": "123456",
  "nickname": "Alice",
  "email": "alice@example.com",
  "phone": "13800000000"
}
```

- 成功响应：

```json
{
  "code": "0",
  "message": "注册成功",
  "data": {
    "userId": "uuid",
    "username": "alice"
  }
}
```

### 5.2 登录
- 方法：`POST`
- 路径：`/api/auth/login`
- 请求体：

```json
{
  "username": "alice",
  "password": "123456"
}
```

- 成功响应：

```json
{
  "code": "0",
  "message": "登录成功",
  "data": {
    "token": "satoken-token",
    "userId": "uuid",
    "username": "alice",
    "nickname": "Alice",
    "role": "user"
  }
}
```

### 5.3 登出
- 方法：`POST`
- 路径：`/api/auth/logout`
- 鉴权：需登录

### 5.4 当前用户
- 方法：`GET`
- 路径：`/api/auth/me`
- 鉴权：需登录

## 6. 用户管理接口（管理员）

### 6.1 分页查询用户
- 方法：`GET`
- 路径：`/api/users`
- 参数：`pageNo`、`pageSize`、`keyword`、`status`、`role`
- 鉴权：管理员

### 6.2 查询用户详情
- 方法：`GET`
- 路径：`/api/users/{userId}`
- 鉴权：管理员

### 6.3 新增用户
- 方法：`POST`
- 路径：`/api/users`
- 鉴权：管理员

### 6.4 更新用户
- 方法：`PUT`
- 路径：`/api/users/{userId}`
- 鉴权：管理员

### 6.5 删除用户
- 方法：`DELETE`
- 路径：`/api/users/{userId}`
- 鉴权：管理员（逻辑删除）

## 7. 个人中心接口

### 7.1 获取个人资料
- 方法：`GET`
- 路径：`/api/profile`
- 鉴权：需登录

### 7.2 更新个人资料
- 方法：`PUT`
- 路径：`/api/profile`
- 鉴权：需登录

### 7.3 修改密码
- 方法：`PUT`
- 路径：`/api/profile/password`
- 鉴权：需登录
- 请求体：

```json
{
  "oldPassword": "123456",
  "newPassword": "new123456"
}
```

## 8. 错误码建议

- `1001`：参数错误
- `1002`：未登录
- `1003`：无权限
- `2001`：账号已存在
- `2002`：账号或密码错误
- `2003`：用户不存在
- `2004`：旧密码错误
